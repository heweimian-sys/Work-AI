# Work-AI 工作 AI 开发总仓库

> 工作 AI 项目存档空间——Agent、MCP、工具链、前后端服务的统一管理。

## 这是什么

本仓库用于记录和开发工作中的 AI 项目，是公司 AI 能力、工具链、Agent、MCP、前后端服务、测试资产和技术文档的总仓库。

本仓库不是单一的"航海资料 Agent"项目。航海小抓、飞书 CLI 等都只是后续可能接入的工作 AI 模块或子项目。

## 仓库结构

```
Work-AI/
├── README.md                    ← 你现在看的这个
├── AGENTS.md                    ← AI 开发助手工作原则
├── 更新记录.md                   ← 仓库整体更新记录
├── 产品需求.md                   ← 仓库级需求池
├── 开发计划.md                   ← 仓库级开发计划
│
├── 项目/                        ← 所有工作项目
│   （后续填入 Agent 项目等，每个项目自带文档和代码目录）
│
└── 知识库/                      ← 所有可复用的东西
    ├── 提示词/                   ← 工作 Prompt / 模板
    ├── 学习笔记/                 ← 工作学习笔记
    └── 技术文档/                 ← 通用技术文档
        └── 代码接入说明.md
```

## 工作方式

每次开发开始前必须先阅读：

1. `README.md`（本文件）
2. `开发计划.md`
3. `更新记录.md`

完成开发后必须更新：

1. `更新记录.md`
2. 必要时更新 `开发计划.md`
3. 与本次改动相关的技术文档或产品需求

## 开发规范

- 文档默认使用中文
- 代码注释使用中文
- Commit Message 使用规范格式（`feat:`、`fix:`、`docs:`、`chore:`）
- 如果发现项目结构不符合规范，先征求确认，不直接重构
