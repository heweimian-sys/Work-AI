# 工作 AI 项目需求池

## 背景

本仓库用于沉淀工作中的 AI 项目需求，统一管理公司 AI 工具链、Agent、MCP、前后端服务和自动化能力。

## 目标

- 记录工作 AI 项目的需求、边界、优先级和验收标准。
- 对不同 AI 模块进行清晰归档，避免个人项目和工作项目混合。
- 支持通过 Agent 或 MCP 工具链复用能力。
- 保留可追溯的执行日志与测试记录。
- 接入飞书 CLI 能力，用于后续对接飞书相关工具链。
- 接入航海小抓能力，作为工作 AI 子模块之一，而不是整个仓库的唯一项目。

## 待明确问题

1. 当前工作 AI 项目有哪些，优先级如何排序？
2. 飞书 CLI 需要解决哪些具体工作流问题？
3. 航海小抓是否只作为资料抓取子模块，还是要扩展成完整 Agent？
4. 各模块是否需要登录、授权、人工审核或数据权限控制？
5. 是否有合规、隐私或访问频率限制？
6. 飞书 CLI 与航海小抓源码当前存放在哪里？

## 飞书周报 Agent

### 业务背景

- 每天记录日志和日报。
- 每周五晚上根据本周日报整理个人周报。
- 每周日根据已确认的个人周报整理部门周报。
- 当前主要问题是忙时漏写日报、周报信息整合困难、内容取舍和格式容易出错。

### 输出结构

- 个人周报：核心工作、项目进展、思考沉淀、下周计划。
- 部门周报：核心工作、常规性事务工作、个人思考、下周工作计划。

### 边界与验收

- Agent 负责候选筛选、归类和表达；确定性脚本负责采集、校验和飞书写入。
- 个人周报必须先确认候选池，部门周报必须以已确认的个人周报为主要输入。
- 不得编造事实、数据、完成状态或业务影响。
- 正式写入前必须展示完整草稿并获得用户对当前版本的明确确认。
- 栏目缺失、存在待确认占位符或目标文档未配置时禁止发布。
