# 代码接入说明

## 当前结论

本工作区当前没有发现飞书 CLI 或航海小抓源码。后续拿到源码后，需要先确认代码职责，再放入对应目录。

`Work-AI` 是工作 AI 开发总仓库，不是单一航海资料 Agent 仓库。任何新代码都必须先判断属于哪个工作 AI 模块，再决定目录位置。

## 接入原则

- 不混入个人仓库，所有公司 AI 项目代码只进入 `Work-AI` 工作仓库。
- 不随意重构原始代码；如发现结构不符合规范，先提出建议并等待确认。
- 接入后必须补充运行说明、环境变量说明和最小验证步骤。
- 如代码包含密钥、Token、Cookie 或内部地址，必须移入 `.env` 或单独的私密配置，不提交到 Git。

## 建议目录

### 飞书 CLI

- 如果代码主要是 MCP 工具、飞书 API 封装或命令行工具，建议放入 `MCP/飞书CLI/`。
- 如果代码主要是后端服务、任务脚本或接口服务，建议放入 `后端/飞书CLI/`。

### 航海小抓

- 如果代码主要体现抓取策略、Agent 流程或提示词编排，建议放入 `Agent/航海小抓/`。
- 如果代码主要是独立爬取程序、任务调度或数据处理服务，建议放入 `后端/航海小抓/`。

## 接入后检查清单

- [ ] 代码目录归位。

- [ ] 补充 `.env.example`。
- [ ] 补充运行说明。
- [ ] 补充最小验证命令。
- [ ] 更新 `开发计划.md`。
- [ ] 更新 `更新记录.md`。
- [ ] 执行 Git Commit。
