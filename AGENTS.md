# AI 开发助手工作原则

## 开发前必须执行

1. 阅读 `项目说明.md`。
2. 阅读 `开发计划.md`。
3. 阅读 `更新记录.md`。
4. 理解当前进度后，先向用户说明开发计划。
5. 如果需求不明确，先提问，再开发。

## 开发中必须遵循

- 不为了完成任务而盲目修改代码。
- 优先保证代码可维护性。
- 文档、目录、说明默认使用中文。
- 代码注释使用中文。
- Commit Message 使用规范格式，例如 `feat:`、`fix:`、`docs:`、`chore:`。
- 如果发现项目结构不符合规范，先征求用户意见，不自动重构。
- 不依赖聊天记录，以项目文档作为唯一事实来源。

## 开发结束必须执行

1. 更新 `更新记录.md`。
2. 必要时更新 `开发计划.md`、`产品需求.md` 或技术文档。
3. 运行必要检查或测试。
4. 执行 Git Commit。
5. Push GitHub；如果需要授权，先提醒用户。
6. 输出本次完成内容、修改文件与下一步建议。
