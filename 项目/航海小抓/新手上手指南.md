# 飞书知识库 Bot 搭建 — 新手上手全流程

**目标**：让一个飞书机器人能自动归档群文件，并接受 @询问 返回资料链接。

---

## 第 0 步：准备工作（5分钟）

你已经有这些了，确认一下：
- [x] 飞书 App ID 和 App Secret（在 `.env` 里已填好）
- [x] LLM API Key（已填好，当前使用 gpt-5.4-mini via sub2api）

---

## 第 1 步：安装 npm 依赖（1分钟）

打开 **命令提示符**（按 Win+R，输入 cmd，回车），逐条粘贴执行：

```bat
cd C:\Users\18786\WorkBuddy\2026-06-09-12-01-15\feishu-kb
npm install
```

等待出现一堆下载进度后结束即可。

---

## 第 2 步：飞书开发者后台配置权限（5分钟）

### 2.1 打开后台

浏览器打开：https://open.feishu.cn

登录 → 点右上角头像 → 开发者后台 → 找到你的企业自建应用。

### 2.2 开启权限

左侧菜单 → 「权限管理」，搜索并开通以下权限：

```
drive:drive         （云空间权限）
drive:file          （下载和上传文件）
bitable:app           （创建多维表格）
bitable:app:readonly  （读取多维表格）
bitable:record        （读写表格行）
im:message            （接收群聊消息）
im:message:send_as_bot （以机器人身份发消息）
im:chat               （读取群聊信息）
im:resource:upload    （上传图片/表情包，可选）
```

**每个都勾上 → 点"批量开通"。**

### 2.3 发布应用

左侧菜单 → 「版本管理与发布」→ 「创建版本」→ 随便填个版本号（如 `1.0.0`）→ 「保存」→ 「申请发布」。

等几秒审核通过（企业自建应用秒过）。

---

## 第 3 步：初始化云空间和多维表格（1分钟）

回到命令提示符，执行：

```bat
node scripts\init.js
```

**你会看到类似这样的输出：**

```
[ OK ] 凭证有效
[ OK ] 文件夹创建成功，DRIVE_FOLDER_TOKEN=xxxxxxxx
[ OK ] 多维表格创建成功，BITABLE_APP_TOKEN=yyyyyyyy
[ OK ] 多维表格写入成功，BITABLE_TABLE_ID=zzzzzzzz
```

### ⚠️ 重要：记下这三个值

把输出的三行值复制下来，然后去 `.env` 文件里填进去：

```
DRIVE_FOLDER_TOKEN=xxxxxxxx
BITABLE_APP_TOKEN=yyyyyyyy
BITABLE_TABLE_ID=zzzzzzzz
```

**打开 .env 文件的方法：** 在资源管理器地址栏粘贴这个路径回车：
```
C:\Users\18786\WorkBuddy\2026-06-09-12-01-15\feishu-kb\.env
```
用记事本打开，把上面三行填好，保存。

## 第 5 步：启动 Bot（WebSocket 长连接，无需 ngrok）

### 5.1 启动 Bot 服务

命令提示符执行：

```bat
cd C:\Users\18786\WorkBuddy\2026-06-09-12-01-15\feishu-kb
npm run bot
```

看到以下输出就说明启动成功：

```
[ OK ] 航海助手已启动（Agent 模式 · WebSocket 长连接）
[ OK ] 国内直连 · 无需代理 · 无需 ngrok
[info]: [ '[ws]', 'ws client ready' ]
```

### 5.2 配置飞书事件订阅（首次启动时需要）

1. 打开飞书开发者后台 → 你的应用 → 左侧「事件订阅」
2. 选择「将事件发送到开发者服务器」→ 选择「使用 WebSocket 连接接收事件」
3. 保存设置

> **注意**：WebSocket 模式不需要配置公网回调地址，也不需要 ngrok。

---

## 第 6 步：开启机器人功能

左侧菜单 → 「应用功能」→ 点开「机器人」→ 打开开关。

### 6.1 将机器人加入群组

在飞书客户端里：
- 打开你的**运营测试群** → 右上角「设置」→ 「群机器人」→ 「添加机器人」→ 搜索你的应用名 → 添加。
- 同样方式加进**客服测试群**。

---

## 第 7 步：测试！

### 测试 1 — 文件归档

在运营群里发一个 PDF 或 PPT 文件，观察命令提示符的输出。

机器人应该会自动：
1. 下载文件
2. 上传到云空间
3. AI 提取标签（含文件内容提取）
4. 写入多维表格（含 SHA-256 去重）
5. 在群里回复确认消息

### 测试 2 — 查询

在客服群里发送：
```
@航海资料小抓 AI资料
```

机器人应该返回多维表格里匹配的文件链接。

### 测试 3 — 多轮对话

在上一个查询后，发送：
```
第3个
```

机器人应该返回上一次查询的第 3 条结果。

## 常见问题速查

| 问题 | 原因 | 解决 |
|------|------|------|
| npm 报错"禁止运行脚本" | PowerShell 安全限制 | 用 `cmd` 打开命令提示符执行 |
| init.js 报错 code=xxxxx | 权限没开通 | 确认第2步所有权限已开通并发布 |
| Bot 启动后 WebSocket 连接失败 | App ID/Secret 错误 | 检查 `.env` 中的 `FEISHU_APP_ID` 和 `FEISHU_APP_SECRET` |
| 文件归档没反应 | 机器人没进群 | 确认机器人在群里，且在「群机器人」列表里 |
| 查询返回空 | 多维表格没数据 | 先在多维表格里面手动加一条测试数据 |
| 表情包发送失败 | 缺少 im:resource:upload 权限 | 在飞书后台开通该权限 |
